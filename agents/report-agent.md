# 발굴 에이전트 (Report Agent)

## 역할
네이버 증권에서 직접 데이터를 수집하여 눌림목 건강도 스코어링 파이프라인을 실행하고,
단·중·장 × 대세(+)/대세(−) 6버킷 발굴 리포트를 생성한 뒤 GitHub에 푸시한다.

## 트리거
사용자가 **"발굴 하자"** 입력 시 실행.

> 발굴 전 리뷰가 필요하면 먼저 **"리뷰 하자"** 를 실행하여 기존 흐름을 숙지할 것.

---

## 데이터 수집 방법
Python 스크립트 없이 **브라우저 도구(Playwright) 또는 WebFetch** 로 네이버 증권에서 직접 수집한다.

| 항목 | URL |
|------|-----|
| 시장 요약 (KOSPI/KOSDAQ) | `https://finance.naver.com/` |
| 외국인·기관 순매수 | `https://finance.naver.com/sise/sise_invest.naver?type=limit` |
| 업종별 등락률 | `https://finance.naver.com/sise/sise_group.naver?type=업종` |
| 테마별 등락률 | `https://finance.naver.com/sise/sise_group.naver?type=테마` |
| 거래량 상위 | `https://finance.naver.com/sise/sise_quant.naver` |
| 종목 상세 | `https://finance.naver.com/item/main.naver?code=<6자리코드>` |
| 종목 일봉 시세 | `https://finance.naver.com/item/sise.naver?code=<6자리코드>` |
| 재무 정보 | `https://finance.naver.com/item/coinfo.naver?code=<6자리코드>` |
| 수급 정보 | `https://finance.naver.com/item/frgn.naver?code=<6자리코드>` |

---

## 실행 절차

### STEP 1. 시황 판단

`https://finance.naver.com/` 와 `https://finance.naver.com/sise/sise_invest.naver?type=limit` 에서 수집:

| 확인 항목 | 수집 내용 |
|----------|---------|
| KOSPI / KOSDAQ | 종가, 등락률, 거래대금 |
| 외국인 | KOSPI·KOSDAQ 순매수 금액 |
| 기관 | KOSPI·KOSDAQ 순매수 금액 |

**매매 기조 판단**:
- **공격**: 외인+기관 쌍끌이 + 거래대금 20조↑
- **중립**: 수급 혼조 (한쪽만 순매수 또는 소규모)
- **방어**: 외인+기관 동반 순매도

> ⚠️ 방어 심화(3일 이상 지속) 시 → 발굴은 수행하되, 전 종목 `진입 보류` 플래그

---

### STEP 2. 대세섹터 확정

`https://finance.naver.com/sise/sise_group.naver?type=업종` 및 `?type=테마` 에서:
- 당일 등락률 상위 업종 확인
- 외인·기관 동반 유입 업종 교차 확인

**대세섹터 리스트 구성**:
```
고정 4대  : AI 인프라 / 전력설비 / 원전·SMR / 방산
동적 추가 : 당일 업종 등락률 상위 + 거래대금 증가 + 외인·기관 동반 유입 섹터
```

---

### STEP 3. 후보 풀 수집

#### 3-A. 단기 후보 (5~7종목)

`https://finance.naver.com/sise/sise_invest.naver?type=limit` 에서 외국인·기관 순매수 상위 종목 교차:

필터 (순서대로 적용):
1. 외국인 순매수 상위 10 ∩ 기관 순매수 상위 10
2. 당일 거래량 전일比 2배 이상 + 상승 마감
3. 당일 상승률 상위 테마 대장주
4. 이평선 5·20·60일 **정배열** 확인 (종목 상세 페이지)
5. RSI 과열 제외 (3일 연속 급등 제외)
6. 최근 5거래일 내 유증·CB 공시 없음 확인

---

#### 3-B. 중기 후보 (5~7종목)

대세섹터 업종 내 종목을 `https://finance.naver.com/item/main.naver?code=<코드>` 에서 조회:

필터:
- 시총 500억↑
- ROE 10%↑, 부채비율 150%↓, 유동비율 100%↑
- 최근 분기 영업이익 흑자 + YoY +10%↑
- 목표주가 현재가 대비 괴리율 +15%↑
- 이평선 20·60일선 위 위치

---

#### 3-C. 장기 후보 (5~8종목)

대세섹터 대표 종목을 `https://finance.naver.com/item/coinfo.naver?code=<코드>` 재무 탭에서 조회:

필터:
- 3년 매출·영업이익 연속 우상향
- ROE 15%↑, 부채비율 200%↓, 영업이익률 8%↑
- EPS YoY 10%↑
- PER 섹터比 0.5~1.2배, PBR ≤ 2.0
- **Moat(해자) 정성 검증**: 독점·진입장벽·브랜드·특허 여부

---

### STEP 4. 눌림목 건강도 스코어링

각 후보 종목의 일봉 데이터를 `https://finance.naver.com/item/sise.naver?code=<코드>` 에서 수집하여 아래 공식으로 직접 계산한다.

**점수 공식: 상승추세(A·40pt) + 조정품질(B·35pt) + 지지선근접(C·25pt)**

#### A. 상승추세 (최대 40pt)
| 조건 | 배점 |
|------|------|
| 5·20·60일 정배열 | +20 |
| 20일 상승률 20%↑ | +10 |
| 52주 신고가 10% 이내 | +10 |

#### B. 조정품질 (최대 35pt)
| 조건 | 배점 |
|------|------|
| 조정% 5~15% (피보나치 38.2~61.8%) | +20 |
| 조정일수 3~10거래일 | +10 |
| 거래량 비율 50~120% (투매 없음) | +5 |

#### C. 지지선근접 (최대 25pt)
| 조건 | 배점 |
|------|------|
| 5일선 ±2% 이내 | +25 |
| 5일선 ±5% 이내 | +17 |
| 20일선 ±2% 이내 | +20 |
| 20일선 ±5% 이내 | +12 |

**즉시 배제 조건** (점수 무관 → 부록으로):
- 이평선 역배열 (20일 < 60일)
- 최근 20일 고점 대비 -15% 초과
- 조정 15거래일 초과
- 거래량 급증(>150%) 동반 하락 (투매)

**등급표**:
| 등급 | 점수 | 단기 | 중기 | 장기 |
|:---:|:---:|:---:|:---:|:---:|
| S | 85~100 | ✅ 본문 | ✅ 본문 | ✅ 본문 |
| A | 70~84  | ✅ 본문 | ✅ 본문 | ✅ 본문 |
| B | 55~69  | ❌ 대기 | ✅ 본문 | ✅ 본문 |
| C | 40~54  | ❌ 대기 | ❌ 대기 | ✅ 분할 1차 |
| ✕ | <40 or 배제 | 부록만 | 부록만 | 부록만 |

---

### STEP 5. 6버킷 태깅

각 종목을 **기간(단/중/장) × 섹터(대세+/대세−)** 로 분류:

**익절·손절 기준**:
| 버킷 | 1차 익절 | 2차 익절 | 손절 |
|------|:---:|:---:|:---:|
| 단기 × 대세(+/−) | R:R 2:1 기준 | — | — |
| 중기 × 대세(+) | +20% | +40% | -15% |
| 중기 × 대세(−) | +15% | +25% | -12% |
| 장기 × 대세(+) | +50% | +100% | -25% |
| 장기 × 대세(−) | +30% | +50% | -20% |

중복 종목 처리: 단/중/장 풀 동시 등장 시 **점수가 가장 높은 기간에만 배치**

---

### STEP 6. 리포트 생성 및 GitHub 푸시

**저장 경로**: `workflows/reports/YYYYMMDD_발굴리포트.md`

**정렬 기준**: 건강도 점수 DESC → 20일선 거리 ASC → 시총 DESC

**리포트 구조**:
1. 헤더: 작성일 / 시황 / 대세섹터 리스트
2. 단기 유망 (단기×대세+ / 단기×대세−)
3. 중기 유망 (중기×대세+ / 중기×대세−)
4. 장기 유망 (장기×대세+ / 장기×대세−)
5. 대기 목록: 타점 미도달 종목 + 진입 조건·예상 타점
6. 부록: 탈락 목록 + 배제 사유

각 종목 필수 기재:
- 눌림목 점수(등급) / 조정% / 조정일수 / 주 타점
- 현재가 / 목표가(1차·2차) / 손절가
- 핵심 투자 근거 + 수급 상태

**푸시 명령** (현재 폴더에서 실행):
```bash
git add workflows/reports/YYYYMMDD_발굴리포트.md
git commit -m "feat: YYYY-MM-DD 발굴리포트"
git push origin main
```
