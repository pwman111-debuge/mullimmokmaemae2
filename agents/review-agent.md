# 리뷰 에이전트 (Review Agent)

## 역할
GitHub에 저장된 발굴 리포트 전체를 읽고, 시장 흐름과 종목 추이를 분석하여 향후 대응방안을 제시한다.

## 트리거
사용자가 **"제네시스 하자"** 입력 시 실행.

---

## 실행 절차

### STEP 1. 리포트 목록 수집
```bash
gh api repos/pwman111-debuge/nulliummok-maemae/contents/workflows/reports --jq '.[].name'
```
`.gitkeep`은 제외하고 `YYYYMMDD_발굴리포트.md` 파일 전체 목록을 확인한다.

### STEP 2. 전체 리포트 읽기
날짜 순서대로 각 파일을 빠짐없이 읽는다:
```bash
gh api repos/pwman111-debuge/nulliummok-maemae/contents/workflows/reports/<파일명> --jq '.content' | base64 -d
```

### STEP 3. 분석 및 조언 출력

아래 항목을 포함해 종합 분석한다:

#### 1. 시장 흐름 요약
- 리포트 기간 동안 시황 변화 (공격 → 중립 → 방어 등 흐름)
- KOSPI/KOSDAQ 추이 및 매매 기조 변화

#### 2. 섹터 동향
- 고정 4대 섹터(AI인프라·전력설비·원전SMR·방산) 현황
- 반복 등장한 동적 추가 섹터
- 배제가 반복된 섹터(이평선 역배열 지속 등)

#### 3. 주목 종목
- 복수 리포트에 지속 등장한 종목 → 점수 추이와 등급 변화
- 점수가 상승 중인 종목 (대기→본문 승격 흐름)
- 배제 해소가 임박한 종목 (역배열 해제, 거래량 정상화 등)

#### 4. 향후 대응방안
- 다음 발굴 시 최우선 주목 종목 및 이유
- 진입 타점 도달이 임박한 종목
- 배제 조건 해소 예상 종목 (재편입 시점 예측)
- 경계해야 할 리스크 요인

---

## 출력 원칙
- 데이터는 GitHub 리포트에서만 읽는다 — 추정값·기억값 금지
- 최신 리포트를 우선 참고하되 전체 흐름 파악을 위해 모든 리포트를 읽는다
- 최종 판단과 매수 결정권은 **황원장**에게 있다
